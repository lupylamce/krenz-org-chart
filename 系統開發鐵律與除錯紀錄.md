## 鐵律清單

### 1. 模組化架構 (S.O.L.I.D.)

| 規則 | 說明 |
|------|------|
| 單一職責 | UI 元件只管渲染。Firebase 邏輯走 `/hooks/` 或 `/services/` |
| 純函數分離 | 考勤計算等商業邏輯必須放 `/utils/`，不可綁死 React 元件 |
| 禁止膨脹 | 新功能必須獨立建檔，**絕對禁止**把元件塞回 `App.jsx` |

---

### 2. Firestore 寫入清洗 (防 `invalid-argument`)

```
// ⛔ 錯誤：undefined 會炸 Firebase
await setDoc(ref, { name: undefined });

// ✅ 正確：送出前過濾
const clean = JSON.parse(JSON.stringify(payload));
await setDoc(ref, clean);
```

### 3. 日期字串處理 (防 Vite 編譯崩潰)

```
// ⛔ 錯誤：正則的 - 會被 Vite 誤判為 Unterminated regex
str.replace(/-/g, '/')

// ✅ 正確
str.split('-').join('/')
```

### 4. 動態考勤數學公式

```
遲到 = 打卡時間 > (表定上班 + 緩衝分鐘)

應下班時間 =
  準時 / 早到  → 表定下班
  緩衝內遲到  → 表定下班 + (實際上班 - 表定上班)
  超過緩衝    → 表定下班 + 緩衝分鐘 (上限)

早退 = 實際下班 < 應下班時間
```

### 5. Canvas 滾輪 vs 彈窗

只要有 Modal / Popup 開啟，滾輪事件 (`wheelHandler`) 必須被攔截，否則組織圖背景會無限縮放。

---

### 6. HTML Date Input 年份防呆

所有 `<input type="date">` 必須加 `max="9999-12-31"`，防止瀏覽器允許輸入超過 4 位的年份 (如 `202622`)。

---

### 7. 組織圖 T 字排版對齊

| 方案 | 結果 |
|------|------|
| ❌ `flex-1` 均分 | 右側幕僚寬度不對稱時，中線偏移 |
| ❌ `grid-cols-[1fr_auto_1fr]` | 大部分情況 OK，但仍有 sub-pixel 偏差 |
| ✅ `grid-cols-2` + `absolute left-1/2 w-[2px]` | **像素級精準**，垂直線用絕對定位畫在正中央 |

---

### 8. 頁面切換必須清理疊加層

Sidebar 每個導覽按鈕的 `onClick` 必須同時：

```jsx
setCurrentView('xxx');
setShowBackend(false);   // 關閉後台名單
setShowCalendar(false);  // 關閉考勤月曆 (若有的話)
```

不這樣做 → Modal 殘留在新頁面上方，UX 災難。

### 10. 加班補休驗證鐵律

| 規則 | 說明 |
|------|------|
| 打卡範圍驗證 | 加班申請的起止時間**必須**落在該日實際打卡 `in ~ out` 之內 |
| 半小時最小單位 | 補休時數以 30 分鐘為最小單位，不足半小時捨去 |
| 到期管理 | 補休效期 6 個月，到期前 30 天提醒 HR，到期後由 HR 決定延後或作廢 |

```js
// ✅ 正確：驗證申請在打卡範圍內
if (applyStartMs < punchInMs || applyEndMs > punchOutMs) {
  return '超出打卡範圍，無法核准';
}
const halfHours = Math.floor(diffMs / (1000 * 60 * 30));
const compHours = halfHours * 0.5;  // 捨去不足 30 分鐘
```

### 11. 假別庫存扣抵鐵律

| 規則 | 說明 |
|------|------|
| 政策與配額分離 | `LEAVE_POLICY` (全域常量) 定義假別規格，`leaveQuota` (員工 Private Data) 存放個人配額 |
| 預設不啟用 | 所有假別預設 `enabled: false`，由 HR 逐一啟用並設定配額 |
| 核准後扣抵 | 假單核准時自動計算並扣抵 `leaveQuota[假別].used`，**非**申請時扣抵 |
